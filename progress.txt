## Codebase Patterns
- gray-matter has no bundled types; a custom declaration lives at src/types/gray-matter.d.ts
- Use `getLogger()` from `../../util/logger.js` for logging (pino-based)
- tsconfig uses `moduleResolution: node` — avoid `/*` globs inside JSDoc block comments (TS parses them as paths)
- Pre-existing test failures exist (9 files, 35 tests) on main — do not block on them
- SkillRegistry tests must mock `os.homedir` (via `vi.mock('os', ...)`) to avoid discovering real global skills
- SKILL.md files must start with `---` as the first line — no content (e.g. HTML comments) before frontmatter
- Executable skill handlers should use Node `child_process` (not Bun.spawn) since vitest runs under Node
